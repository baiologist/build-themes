In the metadata we are showing a title, avatar, and description, dependent on the blog’s global appearance settings, by using {block:ShowAvatar}, {block:ShowTitle} and {block:ShowDescription}.

Pages

Pages come in a couple of varieties:

  • Ask and submit pages
  • Redirects to other websites
  • Standard pages or pages with custom HTML

To add standard pages, redirects and custom HTML pages, go to the customize panel for your blog and select “add a page”. Different page types are accessible from the drop-down menu.

Redirection pages are helpful for sending users to other websites and linking to profiles on social networks. Tumblr sends a HTTP status code of 301, signalling a permanent redirection.

For ask and submit pages, themes need to recognize if the setting is enabled via the dashboard settings panel.

The code for the navigation is:

<div class="nav-wrapper">
  <nav>
    <ul>
        {block:HasPages}
        {block:Pages}
        <li>
          <a href="{URL}">{Label}</a>
        </li>
        {/block:Pages}
        {/block:HasPages}

        {block:SubmissionsEnabled}
        <li>
          <a href="/submit">{SubmitLabel}</a>
        </li>
        {/block:SubmissionsEnabled}

        {block:AskEnabled}
        <li>
          <a href="/ask">{AskLabel}</a>
        </li>
        {/block:AskEnabled}
    </ul>
  </nav>
</div>

The above code is placed after the metadata section.

For pages, the code tests if the user has any pages, has ask enabled and/or has submissions enabled.

Since users choose their own page titles (including for ask and submit pages), variables have been used.

Search

<form action="/search" method="get">
  <input type="text" name="q" value="{SearchQuery}" placeholder="{lang:Search}"/>
</form>

Implementing search only requires a standard HTML form with the value set to the user’s query: {SearchQuery}. The entire search implementation is handled by Tumblr.
