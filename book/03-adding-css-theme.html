The best Tumblr themes are flexible and customizable. Additionally, themes should come with sensible design defaults, as some users do not adjust customizable elements such as the font of a theme.

The CSS for themes which include Tumblr variables or blocks must be kept inline. In that, the CSS must be kept within the HTML document, not in an external CSS file. This is because CSS in external files are not parsed by Tumblr for variables and blocks.

The sample theme has these customizable features:

  1. fonts for the title and body
  2. colors for the masthead and body text and links
  3. background image
  4. support for additional global appearance features

Towards the top of the example theme HTML file, after the meta description element, define the meta elements:

{block:Hidden}
<meta name="font:Title" content="Brutal Type"/>
<meta name="font:Body" content="Sofia Pro"/>
<meta name="color:Masthead text and links" content="{TitleColor}"/>
<meta name="color:Body text" content="#313131"/>
<meta name="image:Background" content=""/>
{/block:Hidden}

{block:Hidden} is used to prevent the meta elements from being rendered on the Tumblr blog’s HTML, as the meta elements are only used by Tumblr in the customize panel.

As discussed earlier, once you have defined the meta elements, and specified the default values within “content”, it is now only a case of referencing the same meta elements in the CSS.

Custom fonts

body {
  font-family: {font:Body};
}
h1, h2, h3, h4, h5, h6 {
  font-family: {font:Title};
}

Implementing the custom fonts with CSS, requires referencing the font meta names from above, with the font-family property.

Custom colors

body {
  color: {color:Body text};
}
.masthead {
  color: {color:Masthead text and links};
}
.masthead a {
  color: {color:Masthead text and links};
}

Again referencing the names from above, and setting the hex CSS color.

Custom image

body {
  background-image: url('{image:Background}');
}

Since we are referencing an external image file, the variable {image:Background}variable should be wrapped within url('').

Global appearance options

body {
  background-color: {BackgroundColor};
}
.blog-title a {
  font-weight: {TitleFontWeight};
}
a {
  color: {AccentColor};
}

As used earlier for the masthead, we can define more global appearance options, this time referencing them in CSS, rather than HTML.

The referencing method for global appearance options is similar to the local appearance options, however since the global appearance options are defined for all Tumblr themes, there is no need to prefix the variable with the type of meta element (for example, the {color:} prefix is not required for {AccentColor}).

Additionally, there is not a requirement to explicitly define the meta element for global appearance options.
